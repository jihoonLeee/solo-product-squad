# 운영 런북

## 배포 절차

1. main 브랜치 기준 빌드 확인
2. 환경 변수 키 존재 확인
3. DB 마이그레이션 실행
4. 애플리케이션 배포
5. 스모크 테스트 수행

## 롤백 절차

1. 직전 배포 버전으로 즉시 되돌림
2. 마이그레이션 영향 확인
3. 장애 공지 갱신

## 장애 대응

- P1: 로그인/대시보드 전체 장애
- P2: 일부 API 오류
- 초기 대응: 최근 배포 여부 확인, 오류 로그 확인, 롤백 판단

## 운영 루틴

- 일간: 에러 추적 확인
- 주간: 지출 입력 성공률과 이탈률 확인
- 월간: 구독 전환율 점검
