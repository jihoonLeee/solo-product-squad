# MVP 스코프

## MVP 목표
사용자가 월간 예산과 고정비를 10분 안에 설정하고 이번 달 잔여 예산을 즉시 확인하게 만든다.

## In-Scope
| 기능 | 우선순위 | 검증 목적 | 완료 기준 |
|------|----------|-----------|-----------|
| 이메일 회원가입/로그인 | Must | 초기 온보딩 허들 검증 | 가입 후 첫 화면 진입 가능 |
| 수동 지출 등록/수정/삭제 | Must | 반복 사용성 검증 | 하루 지출 기록을 1분 내 입력 가능 |
| 월간 예산 설정 | Must | 핵심 가치 검증 | 예산 한도 입력 후 잔여 예산 계산 가능 |
| 카테고리별 소비 요약 | Should | 인사이트 가치 검증 | 주요 카테고리별 비중 확인 가능 |
| 주간 코치 인사이트 | Should | 구독 유지 포인트 검증 | 초과 카테고리와 행동 제안 3개 표시 |

## Out-of-Scope
| 항목 | 제외 이유 | 재검토 시점 |
|------|-----------|-------------|
| 카드/은행 자동 연동 | 초기 개발 복잡도 과다 | 20명 유료 사용자 확보 후 |
| 세금 신고 리포트 | MVP 핵심 가치 아님 | 유지율 확인 후 |
| 가족/팀 공유 | 개인 사용자 우선 | B2B 확장 검토 시 |

## 기술 스택 결정
- 선택: Next.js 풀스택
- 근거: 초기 MVP는 CRUD + 인증 + 대시보드 수준으로 단일 코드베이스가 빠름
- 대안과 기각 사유: Spring Boot 분리는 현재 복잡 정책/배치 요구가 약해 과함

## 마일스톤 일정
| 기간 | 산출물 | 리스크 |
|------|--------|--------|
| 1주차 | UX, 계약, 기본 인증 | 범위 확장 유혹 |
| 2주차 | 지출/예산 기능 | 폼 UX 미흡 |
| 3주차 | 인사이트, QA, 릴리즈 준비 | 데이터 계산 오류 |

## 리스크 및 의존성
- 리스크: 수기 입력 피로도가 커서 유지율이 낮을 수 있음
- 대응: 기본 입력 속도와 반복 카테고리 UX를 우선 최적화
